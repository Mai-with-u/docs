### Participate in Contributions

Welcome to participate in MaiBot development! Before submitting contributions, please first read [Contributing Guide](https://github.com/Mai-with-u/MaiBot/blob/main/docs-src/CONTRIBUTE.md).

Contribution methods:
- **Submit Issue**: Report bugs or suggest features
- **Submit Pull Request**: Fix bugs or implement new features
- **Improve Documentation**: Supplement and improve project documentation
- **Develop Plugins**: Develop third-party plugins for MaiBot

### Issue Feedback

When submitting Issues on GitHub, please:
1. Use clear titles to describe problems
2. Provide reproduction steps and environment information (OS, Python version, MaiBot version)
3. Attach relevant log output
4. Check if there are existing Issues for the same problem

## Design Philosophy

MaiBot's core design philosophy comes from founder SengokuCola:

> This project was originally just to add some extra features to the Niuniu bot, but as more and more features were written, it was finally decided to rewrite. Its purpose is to create a "life form" active in QQ group chats. The goal is not to write a fully functional robot, but a human-like presence that people can perceive as real.

> The program's functional design philosophy is based on one core principle: "Most like rather than good".

> If humans really need an AI to accompany themselves, not everyone needs a perfect "helpful assistant" that can solve all problems, but a "life form" that makes mistakes and has its own perception and thoughts.

# MaiBot 部署概览

**MaiBot 是一个拟人化的智能体**，可以陪你聊天，不断学习了解你，搜集信息，或者用插件或者mcp扩展能力。

MaiBot可以通过本地命令行对话，也可以通过适配器插件连接到不同的IM平台或者任意客户端。

目前**QQ（NapCat）**平台是**使用最多**，**兼容最好**的平台，本教程将会以QQ平台作为使用示例。

## 部署准备

📱 **QQ 小号**（用来登录QQ的NapCat客户端）

🔑 **AI 模型的 API 密钥（需要拥有识图模型和嵌入模型）**

## 部署流程

1. 安装 **MaiBot** 并启动
2. 安装 **适配器** 并填写相关配置
3. 确认 **MaiBot** 与适配器连接成功
4. 配置 AI 模型和 MaiBot 设置 → [配置指南](../configuration/index.md)

## 📦 1.部署 MaiBot 本体

MaiBot 本体 提供 2 种安装方式，任选其一即可：


| 方式                               | 适合人群          |
| -------------------------------- | ------------- |
| [源码安装](./installation.md)        | 快速安装、控制细节 |
| [Docker 部署（文档有待更新）](./docker.md) | 一键部署  |

安装好后，启动MaiBot进行初始化

## 📦 2.安装并配置适配器

要让 MaiBot 连接到即时通讯平台（QQ、微信、Telegram 等）或者其他消息平台，你需要适配器来建立 MaiBot 与这些消息平台的联系。

你可以参考下面的适配器文档来配置对应的适配器，如果你需要快速开始，可以使用 NapCat适配器。

| 适配器 | 支持平台 | 说明 | 类型 |
| --- | --- | --- | --- |
| [NapCat](../adapters/napcat.md) | QQ | 麦麦官方维护的QQ适配器，插件版为最新更新 | 插件版 / 独立版 |
| Snowluma（建设中） | QQ | 新一代 QQ 适配方案，相关文档和能力仍在建设中 | - |
| Telegram | Telegram | Telegram 平台适配器 | - |
| [更多适配器...](../adapters/index.md) | 微信、Discord 等 | - | - |

具体安装和配置步骤请看 [适配器安装](./adapters_installation.md)。

## 📦 3.确认 MaiBot 与适配器连接成功

如果配置正确，当消息平台收到消息时，MaiBot后台会输出一条对应消息的绿色Log

如果消息能够**正确**传递，说明 MaiBot 与适配器**连接成功**

如果没有消息，你需要检查 

**1.适配器是否工作正常** 

**2.适配器与MaiBot的连接是否正常**

（如果使用**插件版**适配器，则不需要关注适配器与MaiBot的链接，只需要关注适配器是否正确连接消息平台即可）
